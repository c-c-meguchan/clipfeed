# Time Profiler の結果を共有する方法

## 「二重で開く」現象について

Time Profiler で計測中に **1枚目はサクサク・2枚目（裏）はラグ** という差が出ている場合、次の可能性があります。

- **1枚目が SwiftUI の Preview**  
  Xcode のキャンバスで **MenuBarView** や **MainPopoverView** の Preview が動いていると、  
  「アプリのポップオーバー」とは別に、Preview 用のウィンドウがもう一つ開いているように見えることがあります。  
  Preview は **ClipboardMonitor やアイコンフラッシュ用の Combine 購読がなく、軽い** ため、そちらだけサクサク動き、  
  **実際のメニューバーから開いたポップオーバー（本番の 1 つ）が 2 枚目として重く感じる**ことがあります。

**確認してほしいこと**

- **Profile 実行時に Xcode の Preview キャンバスを閉じる**（または Preview を止める）  
  → ポップオーバーが 1 つだけになるか確認
- その状態で **メニューバーから開いたポップオーバーだけ** で再コピーして、ラグが再現するか確認

これで「2枚目だけ重い」が消えるなら、**本番の ViewModel / Monitor / 購読まわりの処理**が重い可能性が高いです。

---

## 1.  trace ファイルを保存して渡す（推奨）

1. Instruments で **File → Save**（⌘S）
2. 拡張子 `.trace` のファイルが保存されます
3. そのファイルを共有（ZIP に固めても可）

※ ファイルサイズが大きい場合は、記録時間を「再コピー操作の前後 5〜10 秒だけ」に短くすると小さくなります。

---

## 2. 画面キャプチャで渡す

次の 2 枚のスクリーンショットがあると原因を絞りやすいです。

### A. コールツリー（Call Tree）

- 下の **Call Tree** エリアを表示
- 左の **「Call Tree」** にチェックを入れる
- **「Invert Call Tree」** にチェックを入れる
- **「Separate by Thread」** にチェックを入れる
- **「Hide System Libraries」** にチェックを入れる（自分のコードに集中するため）
- 再コピー操作でラグが出たあと、**メインスレッド（Main Thread）** の行を展開して、**% が大きい行（自分のアプリ名 / ClipboardHistory）** の上から 5〜10 行をキャプチャ

### B. タイムライン

- 再コピー操作した瞬間が分かるように、**タイムライン上でそのあたりを選択**した状態のスクリーンショット

---

## 3. テキストで伝えられること

trace や画像が渡せない場合は、次のように書いてもらえると助かります。

- **メインスレッドで時間がかかっている関数名**（Call Tree で % が大きいものの名前）
- 再コピー操作から画面が反応するまで、**おおよそ何秒かかったか**
